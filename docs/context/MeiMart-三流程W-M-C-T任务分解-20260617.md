# MeiMart 三流程 W-M-C-T 任务分解

> **层级说明**:
> - **W(Week)**: 周单元,8 周完成 MVP
> - **M(Module)**: 业务模块,对应 17 个代码模块
> - **C(Concept)**: 概念/功能点,模块下的逻辑分组
> - **T(Task)**: 具体可执行任务,0.5~2 小时可完成
>
> **三流程并行**: W2 开始,流程 1/2/3 同时进行,W5-W6 联调,W7-W8 收尾上线。

---

## 🗺️ 全局节奏图

```
W1   [共享前置层] (36 个原子任务 / 允许拖到 6-7 天,见 W1 AI 执行版)
       │  ↑ 同步启动:WhatsApp/Google Maps/律师咨询/翻译外包
       ▼ 三流程启动门
       │
W2-W4 ─┬─ [流程1 供给+仓储+浏览(含 i18n 基础)]
       │
W2-W5 ─┼─ [流程2 交易+配送(4 支付 + Google Maps + WhatsApp)]
       │
W2-W4 ─┴─ [流程3 治理+财务]
       │
W5-W6 [全局联调 + UAT 清单 + 真实数据准备]
       │
W7    [端到端验收 + 性能 + 修 bug]
       │
W8    [灰度上线]
```

# 📝 使用说明

1. **执行时**: 拿到任务,从 W1 → W2 → ... 顺序执行,每个 T 完成后 commit
2. **联调时**: W5-W6 的任务需要协调多个流程,建议预留缓冲
3. **跟踪时**: 用 TaskCreate 把每个 T 录入任务系统,实时更新状态
4. **变更时**: 任何 C 或 T 调整,同步更新本文件 + Obsidian memory
