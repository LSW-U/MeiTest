### P0-2：admin cancel order 不触发 refund（用户已付款不退回）

**证据**：
- `admin-order.controller.ts:91-108` cancel 方法调 `orderService.cancelOrderInternal(id, {...})`
- `order.service.ts:481-544` cancelOrderInternal 仅做：释放库存 + Order.status=CANCELLED + 写 OrderEvent(CANCELLED)
- **没有**任何 PaymentService.refund / RefundService 调用

**业务后果**：
- 用户已支付的订单（paymentStatus=PAID）被 admin 取消后，PaymentIntent 仍是 PAID 状态
- 库存退回了，订单取消了，但**钱没退回给用户**
- 这是**资金损失**类的 P0 业务正确性问题

**进度报告承认**：报告 §五 P0 #1 写道"admin orders 取消订单触发 refund：当前 admin cancel 不触发 refund，W5 接入 refund 模块时一起做"——但这是个**业务正确性 P0**，不应推到 W5

**修复方向**（短期 + 长期）：
- **短期（W4 补丁）**：admin cancel 前先检查 `order.paymentStatus === 'PAID'`，若是则抛 `ConflictException({ code: 'E-ORDER-006', message: 'Cannot cancel paid order, please use refund flow' })` 拒绝取消（防资金损失）
- **长期（W5 refund 接入）**：实现 RefundService 后，admin cancel → 自动调 RefundService.create（原路回款 mock），完成后才取消订单

**修复方向选择**：
- 短期方案 0.3 人天，长期方案随 W5 refund 一起做

### P0-4：broadcastInventoryAlert 推送的 `warehouse:${warehouseId}` room 没有自动 join（W3 P0-2 模式复发）

**证据**：
- `realtime.gateway.ts:225` `const room = \`warehouse:${payload.warehouseId}\``
- `realtime.gateway.ts:117-141` handleConnection 自动 join 逻辑：
  ```ts
  if (user.role === 'rider') await client.join(RIDERS_ROOM);
  if (user.role === 'customer_service' || user.role === 'super_admin') await client.join(CUSTOMER_SERVICE_ROOM);
  // warehouse_staff 没有任何自动 join
  ```
- 全代码库 grep `warehouse:` room join：**零**

**结论**：broadcastInventoryAlert 推到 `warehouse:xxx` room，但**没有任何 socket 曾加入这个 room**。W3 整合审查我刚提的 P0-2 模式（customer-service room 无人订阅）**在 W4 又复发了**——这次是 warehouse room。即使 P0-3 修复串接了 broadcastInventoryAlert 的调用点，库存告警 WS 推送仍然完全失效。

**修复方向**：
- handleConnection 加：
  ```ts
  if (user.role === 'warehouse_staff') {
    // 查 user 对应的 WarehouseStaff 记录，拿到 warehouseIds，逐个 join
    const staff = await db.warehouseStaff.findMany({ where: { userId: user.sub }, select: { warehouseId: true } });
    for (const s of staff) await client.join(`warehouse:${s.warehouseId}`);
  }
  ```

**工时估算**：0.3 人天（修代码 + 加 e2e 测试覆盖 warehouse staff 自动 join）
